# 工作原理

在你把 API Key 接入 RovoAPI 之前,你应该先搞清楚一件事:**你的请求经过我们之后,到底发生了什么。**

这不是客套话。你现在要把生产环境的 AI 流量、付费凭据、用户对话内容,全部经过一个第三方服务。跳过这层理解直接抄代码,出问题时你会完全没有排查方向。所以我们从第一原理讲起。

---

## 一句话原理

**RovoAPI 不是一个 AI 模型,是一根管道。**

你发给我们的每一个请求,我们做的事情只有三步:

```
你的代码
   ↓ 发送请求到 api.rovoapi.com
RovoAPI 网关
   ↓ 识别 model 字段,判断该转发给谁
   ↓ 用我们自己的官方账户凭据,转发给对应的上游(Anthropic / OpenAI / Google / DeepSeek)
上游官方 API
   ↓ 原样返回响应
RovoAPI 网关
   ↓ 原样转发回你
你的代码
```

模型怎么"思考"、怎么生成内容,**完全是 Anthropic / OpenAI / Google 在做**。我们不碰这部分。我们做的只是路由和计费这一层。

---

## 为什么这样做有意义?

### 问题 1:你自己直接申请 Anthropic 账户,会遇到什么?

```
新账户 = Tier 1 = 每分钟 50 次请求上限
       中国大陆 IP 被直接拒绝
       需要国际信用卡 + 海外手机号验证
       想要更高额度?需要先充值满 $5000 的历史消费记录
```

### 问题 2:RovoAPI 怎么解决这个?

我们用**自己已经达到高等级的官方账户**去转发你的请求。从 Anthropic 的角度看,这些请求来自一个信誉良好、消费记录充足的账户,而不是一个刚注册的 Tier 1 账户。你不需要自己去"养账户"。

这就是为什么接入 RovoAPI 之后,你的有效速率限制会明显提升 — 不是我们发明了什么黑科技,只是**借用了已经升级过的账户额度**。

---

## 你的数据会被存储吗?

不会。我们是无状态转发:

- 你的请求体(包括对话内容)在转发完成后**不落盘存储**
- 我们**不会**用你的对话内容做任何形式的训练或分析
- 我们记录的日志只包含:时间戳、model 字段、token 用量、HTTP 状态码 — **用于计费和故障排查,不包含你的实际对话内容**

如果你做的是医疗、金融等强合规场景,建议直接联系我们确认数据驻留细节,不要只看这页文档。

---

## 为什么响应格式和官方完全一样?

因为我们**不修改**响应体。Anthropic 返回什么 JSON 结构,我们就原样转发什么 JSON 结构给你 — 除了在 HTTP 头里换成我们自己的认证逻辑。

这意味着:

- 官方 SDK(`anthropic` Python 包、`openai` Node 包)不需要做任何适配,只改 `base_url` 就行
- 官方文档里写的所有字段、所有行为,在 RovoAPI 上都成立
- 我们没有"独家功能"会让你的代码产生官方 API 没有的副作用

**反过来说:** 如果 Anthropic 官方调整了 API 行为或定价,我们也会跟着变 — 我们不能、也不会和上游行为脱节。

---

## 计费是怎么算的?

```
你的请求 → 上游官方按官方价格计费 → 我们原样从你的余额里扣除
```

我们不在官方价格上加价。Claude Sonnet 在 Anthropic 官网是 $3/M 输入,在 RovoAPI 上也是 $3/M 输入。我们的商业模式是赚取规模化运营的效率,不是在你的每次调用上抽成。

失败的请求(网络错误、上游 500、认证失败)**不计费** — 我们只对成功返回内容的请求扣费。

---

## 故障时会发生什么?

如果 Anthropic 官方接口本身宕机,RovoAPI 也会受影响 — 我们没有能力让一个挂掉的上游服务复活。

但如果是**特定路由节点**出问题(比如某个区域的网络链路),我们会自动切换到备用节点重试,这部分对你透明,不需要改代码。

如果你的应用是关键生产环境,建议在代码层面自己加一层模型 fallback(比如 Claude 失败时自动尝试 GPT-4o),这是任何 AI 应用的标准实践,不只针对 RovoAPI。

---

## 什么时候不应该用 RovoAPI?

诚实地说:

- 如果你已经是 Anthropic 的 Tier 4/5 大客户,直连官方可能比经过我们更快(少一跳网络)
- 如果你的合规要求强制必须直连官方 API(不允许任何中间代理),RovoAPI 不适合你
- 如果你只是想试用一两次模型,官方 Console 自带的免费额度可能更直接

RovoAPI 解决的是一个具体问题:**国内/低 Tier 账户访问主流 AI API 时的速率限制和网络稳定性问题。** 如果你不在这个问题里,可能不需要我们。
